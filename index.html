    01 — How it works

    Graph centrality over sentence vectors.

    📥

    Ingest

    Take a transcript from text, a file, or — best-effort — a real YouTube fetch via youtube-transcript-api. Degrades gracefully offline.

    🕸️

    TextRank

    Build a sentence graph with cosine-similarity edges over TF-IDF vectors, then run PageRank power iteration to rank sentences by centrality.

    🏷️

    Keyword + tag

    Rank unigrams/bigrams by aggregate TF-IDF weight and slugify the top terms into auto-tags.

    # the real engine, exactly as shipped
from stash import Store, stash_item

with Store("stash.db") as store:
    result = stash_item(
        store,
        text=open("data/index_funds_talk.txt").read(),
        title="Index funds explained simply",
        num_sentences=3,
        method="textrank",        # or "tfidf_baseline"
    )
    print(result.item.summary)    # extractive, no API key needed
    print(result.item.tags)       # ['index', 'fund', 'broad', ...]

    02 — What's real

    Real engine, optional LLM.

    🧮

    TextRank, implemented

    Cosine over TF-IDF sentence vectors + power iteration of the PageRank recurrence. Plus a TF-IDF top-sentence baseline.

    🗄️

    SQLite store

    Saved items persist url, title, transcript, tags, summary, and saved-at; save/load round-trips are tested.

    🤖

    LLM hook — optional

    llm_summarize only runs if an API key is set; otherwise it returns None and the stored summary is the extractive one. Default behavior is fully offline.

    📝

    Flexible ingestion

    Pass a transcript directly, read it from a file, or fetch a real one from a YouTube id when the network and package are available.

    🧪

    Tested

    pytest suite: TextRank centrality on a constructed hub document, bounded summary length, keyword salience, store round-trip, full offline pipeline.

    📦

    Runs offline

    Committed sample transcripts mean python demo.py works with no network and no key.
